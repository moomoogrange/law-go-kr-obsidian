# 대한민국 법령 Obsidian 위키 (law.go.kr → LLM-native Knowledge Graph)

국가법령정보센터(law.go.kr)에 흩어진 **3,037개 법령 · 6,884개 파일 · 222,279개 상호참조**를
LLM이 그대로 삼킬 수 있는 **Obsidian 볼트 포맷**으로 재구성한 지식베이스입니다.

> **한 줄 요약**: 법령 원문은 공공데이터다. 문제는 "읽을 수 있는 구조"가 아니라 **LLM이 추론할 수 있는 구조**로 가공된 적이 없다는 점이다. 이 저장소가 그 공백을 채운다.

---

## 왜 LLM-native 위키로 만들었나

### 1. 법령 사이트(HTML)는 대한민국 국민을 위한 사이트이다. 

`law.go.kr`은 훌륭한 검색 엔진이지만, 모든 정보가 **HTML + JavaScript 팝업 + PDF + hwps + docs **에 갇혀 있다.

- 법령 상호참조(「산지관리법」→「초지법」→「농지법」)는 **파란 링크**로만 존재 — 그래프 구조가 아니다
- 조문 간 관계, 위임 구조(법률→시행령→시행규칙), 소관부처 변천사는 **사람이 눈으로 읽어야** 추적 가능
- LLM이 RAG로 이걸 다루려면 매번 **크롤링 → 파싱 → 청킹 → 임베딩**을 다시 짜야 한다

### 2. Markdown + Wikilink = LLM의 native 자료구조

Obsidian은 사실상 "LLM이 최적으로 읽는 포맷"을 먼저 만들어둔 도구다.

| Obsidian 요소                        | LLM에게 주는 것                                                           |
| :----------------------------------- | :------------------------------------------------------------------------ |
| `---` YAML frontmatter               | 구조화된 메타데이터 (법령ID · 공포일 · 소관부처 · 상태)                   |
| `[[법령/산지관리법/법률]]` 위키링크  | **조문 간 명시적 엣지** — 그래프 탐색·추론이 1-hop에 끝남                 |
| `tags: 소관부처/농림축산식품부`      | 다차원 패싯 필터링 (부처 × 법령구분 × 상태 × 공포연도)                    |
| 플레인 Markdown                      | 별도 파서·크롤러 없이 `Read`, `Grep`, `Glob`만으로 즉시 질의 가능          |
| 디렉토리 구조 (`법령/{법명}/{위계}`) | 경로 자체가 온톨로지 — 파일명만 봐도 "무슨 법의 어떤 위계 조문"인지 확정됨 |

### 3. 실제 사용 시나리오: Claude Code가 바로 삼킨다

```bash
# Claude Code에게 이 볼트를 컨텍스트로 주면 다음이 즉시 가능:
# - "초지법에서 농지법·산지관리법으로 넘어가는 위임 경로 그려줘"
# - "2013년 이후 농림축산식품부로 이관된 법령 전부 나열해줘"
# - "축사 관련 조문이 언급된 모든 시행규칙 추출해줘"
```

별도 벡터 DB, LangChain 파이프라인, 커스텀 파서 **전혀 필요 없음**.
`Grep "초지법" .` 한 줄이면 222,279개 엣지 중 관련 노드가 다 나온다.

### 4. 사람이 직접 관리해도 이득 (Obsidian으로 열면)

- 그래프 뷰로 법령 생태계 전체 시각화
- Dataview 플러그인으로 "공포연도별 법률 개수" 같은 쿼리가 한 줄
- 백링크 패널이 곧 "이 법을 인용하는 모든 조문" 리스트

## 사용법

### A. Obsidian으로 열기 (인간용)

1. 저장소 clone
   ```bash
   git clone https://github.com/Joseph1981/law-go-kr-obsidian.git
   ```
2. [Obsidian](https://obsidian.md) 실행 → "Open folder as vault" → clone 받은 폴더 선택
3. Graph View 켜기 → 222,279개 엣지가 그려진 법령 생태계 관찰
4. Dataview 플러그인 설치 권장 (각 MOC에서 동적 쿼리 사용)

### B. LLM 컨텍스트로 사용 (Claude Code / RAG)

#### Claude Code에서 직접
```bash
cd law-go-kr-obsidian
claude
> 초지법과 산지관리법의 위임 관계를 그래프로 정리해줘
```

Claude가 `Grep`·`Glob`·`Read`만으로 위키링크를 따라 탐색한다. 임베딩 불필요.

#### 벡터 DB에 적재
- 파일당 YAML frontmatter가 이미 구조화되어 있어 **청킹 경계가 자연스럽다** (조문 단위 또는 파일 단위)
- 위키링크 `[[법령/X/법률]]`은 그대로 엣지 메타데이터로 보존
- 권장 임베딩: 한국어 특화 모델 (`jhgan/ko-sbert-sts`, `BM-K/KoSimCSE-roberta` 등)

### C. grep 기반 즉시 검색

```bash
# "축사" 언급하는 모든 조문
grep -r "축사" 법령/ --include="*.md" -l

# 2024년 이후 제정 법률만
grep -l "공포연도/2024" 법령/ -r
```

## 라이선스 & 저작권

### 법령 원문
대한민국 **저작권법 제7조**에 의해 법령은 저작권 보호 대상에서 제외된다.
따라서 조문 텍스트 자체는 **퍼블릭 도메인**이다.

### 이 저장소의 구조·스크립트·메타데이터
[**CC0 1.0 Universal (퍼블릭 도메인 헌정)**](./LICENSE) — 모든 권리 포기.
출처 표기 없이 상업적 이용·재배포·개변 자유.

> 공공데이터를 LLM 친화 포맷으로 다시 포장했을 뿐이다. 소유권을 주장할 이유가 없다.

## 알려진 한계

- **정지/폐지 법령 포함** — 역사 법령(조선총독부법률, 과거 부처의 옛 부령)도 그대로 유지. `상태` 필드로 필터링 필요.
- **시점 기준 2026-04-01** — 이후 제·개정 법령은 반영 안 됨. 최신성이 중요하면 `law.go.kr`의 Open API를 직접 호출할 것.
- **행정규칙·자치법규 미포함** — 법령(법률·대통령령·부령·규칙) 위주. 고시·훈령·조례·규칙의 일부만 포함.
- **판례 없음** — 조문만 있고 대법원 판례 요지는 링크되지 않음.
